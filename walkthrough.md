# Walkthrough: IT Inventory System — PInventory (Firebase Cloud Migration)

ระบบจัดการสินค้า IT สำหรับโปรเจคสหกิจศึกษา ได้ย้ายฐานข้อมูลสำเร็จรูปจาก SQLite ในเครื่องขึ้นคลาวด์ **Firebase Firestore** (ตามทางเลือกที่ 2) เป็นที่เรียบร้อยแล้ว โดยตัวเซิร์ฟเวอร์ Express.js ยังคงรันและทำหน้าที่คัดกรองข้อมูลให้เช่นเดิม

## 🛠️ รายละเอียดการย้ายฐานข้อมูลไป Firebase Firestore

### 1. โครงสร้างข้อมูล NoSQL บน Cloud Firestore
ฐานข้อมูลได้รับการจัดสรรคีย์และแยกเป็นคอลเลกชัน (Collections) ดังนี้:
* **`users`**: สำหรับบัญชีผู้ใช้ ลิงก์เก็บข้อมูลบทบาท (`admin`/`user`) และข้อมูลโปรไฟล์ส่วนตัว
* **`products`**: เก็บรายละเอียดสินค้า IT ทั้งหมด กำหนดสถานะคลังสินค้า และเก็บ ID ของผู้สร้างที่เป็นความสัมพันธ์กับคอลเลกชัน `users`
* **`purchase_orders`**: เก็บประวัติการซื้อ โดยรายการสินค้าถูกออกแบบให้อยู่ในรูปของ **Nested Array (NoSQL best practice)** ซึ่งจะเก็บไอเทมสินค้าทั้งหมดไว้ในเอกสารใบสั่งซื้อนั้นโดยตรง ลดความซับซ้อนในการสืบค้นข้อมูล
* **`activity_log`**: เก็บบันทึกประวัติการล็อกอินและการปรับเปลี่ยนคลังสินค้าแบบรายครั้ง

### 2. การคำนวณและสืบค้นข้อมูลแบบ In-Memory (ประสิทธิภาพสูง)
เนื่องจาก Firestore เป็นฐานข้อมูล NoSQL ที่ไม่มีการทำ `JOIN` ตารางแบบเดิม:
* ระบบ Backend ได้รับการปรับแต่งให้ทำการสืบค้นข้อมูลแบบประสานร่วมกัน (In-memory grouping & map-reduce) เพื่อคำนวณสถิติยอดเงินสะสม, การจำแนกประเภทสินค้าสำหรับกราฟ และประวัติผู้ใช้งานหน้าแรก (Dashboard) ทำให้รันงานได้เสถียร รวดเร็ว และไม่ต้องเสียเวลาสร้าง Index ค้นหาที่ซับซ้อนบนคลาวด์

## 🎨 การปรับปรุง UI แดชบอร์ด & ระบบอนิเมชั่นเปลี่ยนหน้า (UI Improvements & Transitions)

ระบบได้รับการติดตั้งธีมการเคลื่อนไหวและแผงควบคุมสถิติแบบมือโปรเพิ่มเติมดังนี้:

### 1. ระบบ Watchlist การกระจายสินค้าแบบการ์ดเทรด (Trading Watchlist Cards)
- **การ์ดดีไซน์ระบบสว่างพรีเมียม (Premium Light Theme Alignment)**: ออกแบบใหม่ให้กลมกลืนกับส่วนอื่นของหน้าแดชบอร์ด (การ์ดพื้นหลังขาว/เทา Slate 50 พร้อมเงาแบบนุ่มนวลและฟอนต์สีเข้มอ่านง่าย) ลดการตัดกันของโทนสีที่สว่างและมืดจนเกินไป
- **แท็บควบคุมสถิติ (Interactive Watchlist Tabs)**:
  - **Watchlist**: แสดงผลหมวดหมู่ตามปกติ
  - **Trending**: จัดเรียงหมวดหมู่ตามความเคลื่อนไหวมากที่สุด (Absolute % Change)
  - **Top Gainers**: คัดกรองและแสดงเฉพาะหมวดหมู่ที่มีจำนวนสินค้าเพิ่มขึ้น
  - **Top Losers**: คัดกรองและแสดงเฉพาะหมวดหมู่ที่มีจำนวนสินค้าลดลง
- **แท็บตัวกรองหมวดหมู่ (Sub-category Filters)**: ผู้ใช้สามารถกดเลือกคัดกรองแสดงเฉพาะบางหมวดหมู่ได้แบบ Real-time
- **กราฟ Sparkline สีสันตามทิศทาง**: แสดงกราฟเส้นแนวโน้มย่อย (Mini Sparkline Area Chart) ด้วยสีเขียว Emerald เมื่อเพิ่มขึ้น และสีแดง Rose เมื่อลดลงแบบสม่ำเสมอไม่ขยับสั่นขณะเรนเดอร์ใหม่
- **ปุ่มเพิ่มสินทรัพย์**: เชื่อมโยงตรงไปยังหน้าจัดการคลังสินค้า

### 2. อนิเมชั่นเปลี่ยนหน้าแบบสมูทและรวดเร็ว (Optimized Page Fade Transitions)
- **อนิเมชั่น Fade Snappy**: ปรับปรุงเอฟเฟกต์การสลับหน้าให้มีความคล่องตัว (ใช้เวลา 0.22 วินาที, TranslateY 6px) และลบส่วนของ Blur ฟิลเตอร์ออกเพื่อลดการหน่วงการประมวลผลของหน่วยประมวลผลกราฟิก (GPU Stuttering) ช่วยให้เว็บรู้สึกนุ่มนวลและเร็วขึ้นอย่างเห็นได้ชัด
- **การติดตั้งที่ยืดหยุ่น**: รันผ่าน React Key Re-rendering ตาม `location.pathname` บนเลย์เอาต์หลักอย่างมีประสิทธิภาพ

### 3. ระบบจัดการงานซ่อมบำรุงในแท็บขวา (Maintenance & Repairs Panel)
- **สลับข้อมูลแบบแท็บย่อย (Sub-tab Content Switch)**: ปรับปรุงการ์ดสภาพสินทรัพย์ด้านขวา (IT Asset Overview) ให้แสดงผลเป็น 3 แท็บย่อยเพื่อความเป็นระเบียบและสวยงาม:
  - **Overview**: สถิติสัดส่วนสภาพสินทรัพย์เดิม (Normal, In Use, Under Repair, Broken) ในแบบกราฟแถบเปอร์เซ็นต์
  - **Repairs**: ดึงข้อมูลรายการสินค้าที่มีสถานะชำรุด (Broken) หรือส่งซ่อม (Under Repair) จากฐานข้อมูลโดยตรง พร้อมปุ่ม **"ซ่อมเสร็จ" (Mark Repaired)** เพื่อเปลี่ยนสถานะสภาพสินค้ากลับเป็นปกติ (`Normal`) บนฐานข้อมูลจริง
  - **History**: ประวัติการซ่อมแซมสินทรัพย์ที่แก้ไขเรียบร้อยแล้ว แสดงชื่อผู้ที่ทำการซ่อมและวันเวลาล่าสุดอย่างครบถ้วน

### 4. ปรับเปลี่ยนดีไซน์ป้ายแจ้งเตือนความสำเร็จ (Optimized Toast Notifications)
- **ปรับขนาดและตำแหน่งการแสดงผล**: เปลี่ยนพิกัดตำแหน่งจากเดิมที่ทับซ้อนและยืดขยายสูงยาวกลางจอภาพ (เนื่องจากการชนกันของสไตล์ `top` และ `bottom`) ให้ไปแสดงผลที่ **กึ่งกลางขอบบน (Top Center)** ใต้แถบเมนูนำทาง (Navbar) เล็กน้อย
- **ดีไซน์ทรงเม็ดยาพรีเมียม (Pill Shape)**: ปรับขนาดกล่องแจ้งเตือนให้กระชับมีความโค้งมนสวยงาม (`border-radius: 30px`) และปรับแต่งสีพื้นหลังให้อ่อนบางละมุน (Soft Translucent background) ด้วยสีเขียวพาสเทลอ่อน (#ecfdf5) และตัวหนังสือเขียวเข้ม (#065f46) พร้อมขอบสีเขียวอ่อน พร้อมป๊อปอัพลงมาอย่างนุ่มนวลจากด้านบน (`slideInDown` animation)

### 5. ปรับปรุงหน้า IT Support (Knowledge Base)
- **เนื้อหาใหม่ที่เป็นประโยชน์**: เพิ่มคู่มือการทำงานแบบเป็นรูปธรรม 2 ฉบับ:
  1. **วิธีติดตั้งและตั้งค่าการใช้งาน Zoiper**: ขั้นตอนการติดตั้งและเซ็ตค่า VoIP โทรศัพท์สายใน
  2. **วิธีติดตั้งและตั้งค่าการเชื่อมต่อใช้งาน VPN**: คู่มือติดตั้ง Cisco AnyConnect เพื่อเข้าระบบเครือข่ายบริษัท
- **Timeline Stepper ดีไซน์สวยงาม**: ปรับเปลี่ยนรูปแบบรายการจาก Accordion ข้อความธรรมดา ให้กลายเป็นการ์ดช่วยเหลือขนาดใหญ่ 2 ใบในแบบ Grid Layout
  - แต่ละการ์ดมีหัวข้อ รายละเอียด และขั้นตอนแสดงในรูปแบบ **Visual Timeline Stepper** (มีวงกลมตัวเลข 01, 02, 03... เชื่อมต่อด้วยเส้นตรงแบบมีดีไซน์)
  - ฝัง **สไลด์นำเสนอจาก Canva (Interactive Canva Embeds)** เข้าไปในตัวการ์ดโดยตรงผ่านไอคอน iframe เพื่อแสดงงานนำเสนอต้นฉบับของผู้ใช้แบบเรียลไทม์ พร้อมปุ่มกดเลื่อนสไลด์ที่สวยงาม:
    - สไลด์สำหรับ Zoiper: `https://www.canva.com/design/DAHI-Rb3UQw/view?embed`
    - สไลด์สำหรับ VPN: `https://www.canva.com/design/DAHI9qtPlSQ/view?embed`
  - แทรก **กล่องโค้ดทางเทคนิค (Technical Details Codebox)** ในขั้นตอนที่มีการป้อนโฮสต์เซิร์ฟเวอร์ (`vpn.itopplus.com` และ `voip.itopplus.com`) เพื่อความสะดวกในการคัดลอก
  - ติดตั้ง **ปุ่มคัดลอก (Copy Button)** พร้อมไอคอน ที่ช่วยให้ผู้ใช้งานคลิกคัดลอกค่าโฮสต์ไปใช้ได้ทันที และระบบจะแสดงแถบ Toast แจ้งเตือนคัดลอกสำเร็จด้วยความสวยงาม

### 6. ระบบยืม-คืนอุปกรณ์ไอทีสำหรับพนักงาน (IT Equipment Loan System)
- **ระบบย่อยฝั่งเซิร์ฟเวอร์ (`loans.js`)**: พัฒนา API `/api/loans` เพื่อจัดการกระบวนการยืม-คืนสินทรัพย์บนฐานข้อมูล Firestore:
  - `POST /api/loans/request`: ส่งคำขอยืมอุปกรณ์ (เช็คสถานะการยืมซ้ำซ้อนและโควตาในคลังอัตโนมัติ)
  - `POST /api/loans/:id/return`: ส่งคำขอคืนอุปกรณ์เครื่องที่กำลังยืมอยู่
  - `POST /api/loans/:id/approve` (เฉพาะไอที): ยืนยันรับเรื่องการยืม (จะทำการตัดสต็อกสินค้าในคลังลบ 1 ชิ้นอัตโนมัติ) หรือยืนยันรับเรื่องการคืน (จะเพิ่มสต็อกครุภัณฑ์คืนกลับเข้าคลังบวก 1 ชิ้นอัตโนมัติ)
  - `POST /api/loans/:id/reject` (เฉพาะไอที): ปฏิเสธคำขอการยืม-คืนอุปกรณ์
- **หน้าส่วนติดต่อผู้ใช้ของพนักงาน (`LoansPage.jsx`)**: หน้าจอระบบแยกเฉพาะพรีเมียมสีฟ้า-น้ำเงิน แสดงเมนูนำทางในแถบข้างด้านซ้าย (Main Sidebar):
  - **แท็บอุปกรณ์ที่สามารถยืมได้ (Borrow Equipment)**: แสดงรายการอุปกรณ์ไอทีทั้งหมดที่เป็น "In Stock" และมีสต็อกคงเหลือ แสดงชื่อ ยี่ห้อ รุ่น และจำนวนที่พร้อมให้ยืมอย่างเรียบง่าย เข้าใจง่าย
  - **แท็บรายการยืมของฉัน (My Loans)**: แสดงตารางการทำรายการของตนเองพร้อมป้ายสถานะ (รออนุมัติการยืม, กำลังยืมใช้งาน, รออนุมัติการคืน, คืนเรียบร้อยแล้ว, ปฏิเสธคำขอ) และปุ่มกด **"คืนเครื่อง (Return)"** เมื่อพนักงานใช้คอมพิวเตอร์เสร็จสิ้น
  - **แท็บอนุมัติคำขอ (IT Approval Portal - เฉพาะไอที)**: ตารางรายการรออนุมัติแบบเรียลไทม์เพื่อให้ผู้ดูแลระบบตรวจสอบ กด Approve หรือ Reject ได้ทันที
