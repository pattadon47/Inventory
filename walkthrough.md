# Walkthrough: IT Inventory System — PInventory (Firebase Cloud Migration)

ระบบจัดการสินค้า IT สำหรับโปรเจคสหกิจศึกษา ได้ย้ายฐานข้อมูลสำเร็จรูปจาก SQLite ในเครื่องขึ้นคลาวด์ **Firebase Firestore** (ตามทางเลือกที่ 2) เป็นที่เรียบร้อยแล้ว โดยตัวเซิร์ฟเวอร์ Express.js ยังคงรันและทำหน้าที่คัดกรองข้อมูลให้เช่นเดิม

---

## 🛠️ รายละเอียดการย้ายฐานข้อมูลไป Firebase Firestore

### 1. โครงสร้างข้อมูล NoSQL บน Cloud Firestore
ฐานข้อมูลได้รับการจัดสรรคีย์และแยกเป็นคอลเลกชัน (Collections) ดังนี้:
* **`users`**: สำหรับบัญชีผู้ใช้ ลิงก์เก็บข้อมูลบทบาท (`admin`/`user`) และข้อมูลโปรไฟล์ส่วนตัว
* **`products`**: เก็บรายละเอียดสินค้า IT ทั้งหมด กำหนดสถานะคลังสินค้า และเก็บ ID ของผู้สร้างที่เป็นความสัมพันธ์กับคอลเลกชัน `users`
* **`purchase_orders`**: เก็บประวัติการซื้อ โดยรายการสินค้าถูกออกแบบให้อยู่ในรูปของ **Nested Array (NoSQL best practice)** ซึ่งจะเก็บไอเทมสินค้าทั้งหมดไว้ในเอกสารใบสั่งซื้อนั้นโดยตรง ลดความซับซ้อนในการสืบค้นข้อมูล
* **`activity_log`**: เก็บบันทึกประวัติการล็อกอินและการปรับเปลี่ยนคลังสินค้าแบบรายครั้ง

### 2. การคำนวณและสืบค้นข้อมูลแบบ In-Memory (ประสิทธิภาพสูง)
เนื่องจาก Firestore เป็นฐานข้อมูล NoSQL ที่ไม่มีการทำ `JOIN` ตารางแบบเดิม:
* ระบบ Backend ได้รับการปรับแต่งให้ทำการสืบค้นข้อมูลแบบประสานร่วมกัน (In-memory grouping & map-reduce) เพื่อคำนวณสถิติยอดเงินสะสม, การจำแนกประเภทสินค้าสำหรับกราฟ และประวัติผู้ใช้งานหน้าแรก (Dashboard) ทำให้รันงานได้เสถียร รวดเร็ว และไม่ต้องเสียเวลาสร้าง Index ค้นหาที่ซับซ้อนบนคลาวด์

## 🚀 การรันระบบและยืนยันการทำงาน (Launch & Verify)

ระบบพร้อมให้บริการผ่านสถาปัตยกรรมแบบ Full Stack แล้ว:

1. **Backend Server**: รันสำเร็จและเชื่อมไปยังคลาวด์ Firestore ผ่านพอร์ต [http://localhost:5000](http://localhost:5000)
2. **Frontend Dev Server**: รันสำเร็จและพร้อมเข้าใช้งานที่พอร์ต [http://localhost:5173](http://localhost:5173)

### 📝 คำสั่งจัดการในอนาคต (Terminal Commands):
* **หากต้องการรีเซ็ตหรือสุ่มข้อมูลลงคลาวด์ใหม่**: รันคำสั่ง `node database/seed.js` ในโฟลเดอร์ `server` (ระบบจะลบข้อมูลคอลเลกชันเก่าและทำการใส่ข้อมูลจำลองชุดเริ่มต้นให้ทันที)
* **หากต้องการรันโปรแกรมใหม่**:
  * สตาร์ทเซิร์ฟเวอร์ Backend: `cd server && npm start`
  * สตาร์ทหน้าเว็บ React: `cd client && npm run dev`
